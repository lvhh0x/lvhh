## 2. 사용자가 제기한 문제

현실 출고에서는 **사이즈가 다른 제품을 한 박스에 마구 섞는 걸 선호하지 않는다.**
빈 공간이 다른 사이즈로 **딱 떨어지게 채워지면** 같이 넣어도 되지만, **딱 안 떨어지고 어중간하게 남으면** 사이즈별로 분리해서 박스를 나누는 게 낫다.

## 3. 확정된 규칙 (합의 완료)

### 3.1 핵심 규칙

> **각 제품을 인박스 분해한다.**
> **사이즈별로 "꽉 찬 아웃박스(145×4 = 60단위)"는 먼저 빼서 확정한다.**
> **남은 부분(<60단위)들끼리, 합쳤을 때 정확히 60단위로 딱 떨어지면 한 아웃박스로 합친다(섞음 허용).**
> **딱 안 떨어지면 합치지 않고 사이즈별로 각자 박스(아웃박스/택배/낱개)에 담는다.**

### 3.2 단위 정의 (재확인 — 매우 중요)

| 인박스 | 아웃박스 단위 | 택배박스 단위 |
|--------|---------------|---------------|
| 145 | 15 | 5 |
| 95 | 10 | 3 |
| 60 | 6 | 2 |
| **박스 총용량** | **60** | **12** |

⚠️ **잔여(부분) 처리 시 택배박스 판정은 반드시 "택배박스 단위(총 12)"로 계산한다.**
(이전 검증에서 아웃박스 단위 35와 택배박스 단위 8을 혼동한 오류가 있었음. 아래 3.4 참조.)

### 3.3 사용자 확정 예시 (테스트 케이스)

제품 인박스 용량:
- 40×300: 145→30개, 95→20개, 60→10개
- 60×300: 145→20개, 95→10개
- 110×300: 145→10개

| # | 입력 | 사이즈별 인박스 분해 | 합산 단위 | 결과 | 비고 |
|---|------|---------------------|-----------|------|------|
| A | 40×300 ×30 + 60×300 ×60 | 40: 145×1(15) / 60: 145×3(45) | 60 딱 떨어짐 | 아웃박스 1개 (섞음) | 빈칸 0 |
| B | 40×300 ×60 + 60×300 ×40 | 40: 145×2(30) / 60: 145×2(30) | 60 딱 떨어짐 | 아웃박스 1개 (섞음) | 빈칸 0 |
| C | 40×300 ×90 + 60×300 ×30 | 40: 145×3(45) / 60: 145×1+95×1 | 안 떨어짐 | **40: 아웃박스 1개(부분) / 60: 택배박스 1개** | 분리 |
| D | 40×300 ×210 + 60×300 ×100 | 40: 145×7(105) / 60: 145×5(75) | 합 180 = 60×3 딱 | 아웃박스 3개 | |
| E | 40×300 ×180 + 60×300 ×40 | 40: 145×6(90) / 60: 145×2(30) | 합 120 = 60×2 딱 | 아웃박스 2개 | |

### 3.4 케이스 C의 60 처리 (택배박스 확정)

60×300 ×30 = 145×1 + 95×1.
- 택배박스 단위: 145=5, 95=3 → 합 **8 ≤ 12** → **택배박스 1개에 들어감.** ✅
- (아웃박스 단위로는 15+20=35지만, 잔여 판정은 택배 단위로 해야 함.)

→ 케이스 C 정답: **40은 아웃박스 1개(부분, 145×3), 60은 택배박스 1개(145×1+95×1).**

## 4. 미확정 질문 (새 세션에서 사용자에게 최종 확인할 것)

### ❓ 질문 1 — 케이스 "40×90 + 60×120"의 숫자 확인

논의 중 사용자가 *"60이 100이 아니라 120이면 40 아웃박스 2개, 60은 아웃박스 1 + 택배 1"* 이라고 예시를 들었는데,
40×300 ×90 = 145×3 = **아웃박스 1개(부분)** 이 되어 "40 아웃박스 2개"와 숫자가 맞지 않는다.

**→ 확인 필요:** 이 예시에서 40 수량이 90이 맞는지, 아니면 다른 값(예: 120 이상)이었는지.
40 수량을 정확히 확정해야 이 시나리오를 테스트 케이스로 넣을 수 있다.

### ❓ 질문 2 — 3종류 이상 제품의 "부분 합치기" 조합 규칙

제품이 3개 이상이면(특히 DB 연동 후 제품이 많아짐) 부분 박스들의 합치는 조합 경우의 수가 늘어난다.

예: 40 부분(45) + 60 부분(15) + 110 부분(30)이 남을 때 → 어떤 둘을 합쳐야 60이 되는가?

**제안(단순·저오류·예측가능 우선):**
> "전체 부분의 합이 60의 배수면 **큰 사이즈부터 순서대로** 채워 60씩 묶고, 아니면 전부 사이즈별 분리."
> 복잡한 부분집합 탐색(subset-sum)은 제품이 많아질수록 위험하므로 채택하지 않는다.

**→ 확인 필요:** 이 단순 규칙으로 충분한지, 아니면 더 정교한 조합을 원하는지.

## 5. 설계 방향 (제안 — 확정 후 구현)

사용자 요구("제품 종류 많아짐 + 단순 + 저오류 + 최적 간단")를 반영한 제안:

> **분리 포장 알고리즘 = "제품(사이즈)별로 독립 처리 + 부분만 딱 떨어질 때 합치기"**
>
> 1. 각 제품 → 인박스 분해 (엔진① 그대로 사용)
> 2. 사이즈별로 꽉 찬 아웃박스(60단위) 먼저 확정해서 빼기
> 3. 남은 부분들의 합이 60의 배수인지 검사
>    - 배수면 → 큰 사이즈부터 60씩 채워 아웃박스로 합침 (기존 FFD 합산과 동일 결과)
>    - 아니면 → 사이즈별로 각각 잔여를 **택배(12) → 낱개 → 아웃박스(부분)** 규칙으로 따로 처리
>
> ※ 잔여 판정은 **택배박스 단위(총 12)** 로 한다 (3.4의 오류 재발 방지).

이 구조의 장점:
- 제품이 N개로 늘어도 각자 독립 처리 → 경우의 수 폭발 없음
- "딱 떨어질 때만 합친다"는 단일 조건 → 버그 여지 적음
- 기존 엔진②의 잔여 처리(택배/낱개) 규칙을 **사이즈 그룹별로 재사용**
